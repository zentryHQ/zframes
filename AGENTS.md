# zframes

AI-personalizable market dashboard framework (working name — do NOT call it "hyperframes"; that's HeyGen's project, which we use as the distribution-model reference). An agent reads the frame catalogue (Zod → JSON Schema), emits a `dashboard.json` spec, and the runtime validates + renders it as a live dashboard.

## Commands

```bash
pnpm install
pnpm dev          # runtime at http://localhost:37263 (project default port, strict)
pnpm typecheck    # tsc --noEmit across all workspace packages (pnpm -r)
pnpm build        # vite build of the runtime
pnpm build:cli    # build the zframes CLI + its prebuilt runtime bundle
pnpm zframes serve <dashboard.json>   # the runtime: serve a dashboard live (--port to change)
```

## Structure

- `packages/core` — `defineFrame`/`defineFrameMeta`, registry, `DashboardSpecSchema`, `DashboardRenderer` (CSS-grid; error cards for invalid config AND missing capabilities), capability-routed multi-provider hooks, `catalogueForAI`. The card/error/chrome rendering both renderers share lives in `frame-content.tsx` (`FrameContent` + `FRAME_CSS`). Spec extras: `background` (`BackgroundSchema`) on the dashboard, plus four dashboard-wide cosmetic groups split by fault line — `grid` (geometry: columns/rowHeight/gap), `theme` (`ThemeSchema`: colour — accentHue + accentSat for the accent, baseHue + baseSat for the dark card-surface tint, upColor + downColor for the semantic gain/loss hex pair routed through `--zf-up`/`--zf-down`), `typography` (`TypographySchema`: fontFamily sans/mono/serif + numericStyle proportional/tabular + scale global text size, applied as the host root font-size) and `appearance` (`AppearanceSchema`: card-surface radius/borderStrength/surfaceOpacity/density/elevation), each field mapped to a `--zf-*` var read by `FRAME_CSS` (the base-surface gradient is expressed off `--zf-base-hue`/`--zf-base-sat` with lightness baked, and the family routes through the `--font-dmsans` token, so the defaults reproduce the original look to within ~1/255). Named one-click looks live in `presets.ts` (`THEME_PRESETS`, exported from `@zframes/core`) — pure data the editor's Cosmetics rail and the agent can apply, never a separate render path. Every default is a visual no-op, and a legacy `grid.radius` auto-migrates into `appearance` via a spec `preprocess`. Frame meta carries a required `category` (one of `FRAME_CATEGORIES`, an ordered `{key,label,description}` taxonomy exported from `@zframes/core` — it groups the editor palette into labelled sections and is surfaced in `catalogueForAI` so the generating agent reasons by family), optional `chrome: "bare"` (renders chrome-less — headings become zone dividers) and optional `layout` (`{w,h,minW?,minH?,maxW?,maxH?}` — editor sizing only, ignored by the CSS-grid renderer). Deep exports (`@zframes/core/spec`, `/frame`, `/catalogue`) are React-free for tooling. Opt-in subpaths back the editable dashboard + the runtime: `@zframes/core/editor` (`DashboardEditor` — see Conventions), `@zframes/core/serve` (Node-only `handleSpecRead`/`handleSpecWrite` + `handleProxy` — the spec read/write contract AND a host-allowlisted same-origin official-data proxy at `/__zframes/proxy`, all shared by the dev plugin and the CLI), and `@zframes/core/vite` (`dashboardWriteback()` dev plugin, which also serves the spec for the app to fetch). Provider primitives, React-free and used by every provider: `@zframes/core/fetch` (the shared transport — UA, proxy rewrite, abort, optional Zod validation) and `@zframes/core/cache` (`TtlCache` — the shared TTL + in-flight-dedup + stale-on-error + opt-in-persist cache; see the caching convention below). **Official-data proxy:** browsers can't fetch CORS-blocked official sources (SEC XBRL, H.15, OFR, halts), so a provider passes `fetchJson(url, schema, { proxied: true })` — in the browser this rewrites to the same-origin `/__zframes/proxy?url=…` route (Node relays it; allowlisted hosts only, https/GET-only, browser UA or `--contact`/`ZFRAMES_CONTACT`); in Node it's a no-op (no CORS there). Proxied frames degrade to empty on a static host with no runtime. See the proxy decision in `docs/decisions/providers/providers.md`.
- `packages/charts` — D3 base chart layer ported from zTerminal (`tree-chart`, `heatmap-chart`, `multi-series-line-chart`, `stacked-area-chart`, `pie-chart`, `mini-line-chart`) plus `theme.css` (zTerminal design tokens for Tailwind v4). Implementation-agnostic: no business logic, no data fetching.
- **Crypto market data (keyless):**
  - `packages/provider-hyperliquid` — free no-key provider: allMids WS (one subscription per HIP-3 dex, added lazily from requested symbols), day stats per dex (full default-dex universe when no symbols given; a `"<dex>:*"` wildcard symbol — e.g. `"xyz:*"` — returns that dex's *entire* universe, used by the ticker tape to surface every HIP-3 equity), fundingHistory, candleSnapshot, open interest (`open-interest` capability — live single-venue notional, read from the same `metaAndAssetCtxs` call as day stats; no history endpoint)
  - `packages/provider-coingecko` — global marketcap + dominance (`global-market`) and per-coin market caps (`coin-markets`, top-50 by mcap with 24h change); free tier no key, both endpoints cached behind `TtlCache` (stale-on-429)
  - `packages/provider-coinpaprika` — broad multi-window coin movers across ~2000 coins (`coin-movers` capability)
  - `packages/provider-alternativeme` — fear & greed index (`sentiment` capability)
- **On-chain / DeFi (keyless):**
  - `packages/provider-defillama` — keyless via api.llama.fi (CORS-open): TVL per chain (`tvl`), DEX volume (`dex-volume`, snapshot + per-protocol history), protocol TVL (`protocol-tvl`, snapshot + per-protocol history), protocol fees (`protocol-fees`). Snapshots/histories cached behind `@zframes/core/cache`'s `TtlCache`
  - `packages/provider-mempool` — Bitcoin fees, mempool, blocks, hashrate, difficulty, mining pools, and Lightning stats (keyless, mempool.space)
- **Derivatives / options (keyless):**
  - `packages/provider-deribit` — BTC/ETH put-call ratio, OI-by-strike, DVOL volatility index (Deribit public API, no key)
- **Official US macro & financial data (keyless, same-origin proxy required — these degrade to empty on a static host):**
  - `packages/provider-treasury` — U.S. Treasury data: average interest rates, debt-to-penny, completed auctions, and the daily yield curve (fiscaldata + home.treasury.gov, both proxied)
  - `packages/provider-nyfed` — New York Fed reference rates (SOFR, EFFR, repo)
  - `packages/provider-ofr` — OFR Financial Stress Index (`financial-stress` capability)
  - `packages/provider-bls` — BLS public time series (CPI, unemployment, etc.)
  - `packages/provider-finra` — FINRA daily reported short-sale volume (`short-volume`; cdn.finra.org is CORS-blocked in the browser, so it routes through the proxy)
  - `packages/provider-sec` — SEC EDGAR company filings (submissions endpoint is CORS-safe; the companyfacts/XBRL blob routes through the proxy)
  - `packages/provider-news` — news-outlet headlines via RSS (`news` capability)
- **Keyed / account tier (opt-in, not in the published keyless CLI by default):**
  - `packages/provider-binance` — connected Binance account (`portfolio` capability); secret never reaches the browser — routed through the runtime's server-side signed relay
  - `packages/provider-wallet` — public on-chain wallet address (`portfolio` capability); Ethereum balances via public RPC + bundled token list, priced through CoinGecko; no key, no relay
- `packages/frames` — 76 frames across 12 `FRAME_CATEGORIES` families. **`src/schemas.ts` + `allFrames` in `src/index.ts` are authoritative** — pure Zod, no React, imported by both the components and the CLI. New frame = add meta to schemas.ts + component file + `allFrames`. Families (matching `FRAME_CATEGORIES` keys):
  - `markets` — Prices & Markets: `price-chart` (liveline candle/line, the centerpiece), `price-compare`, `price-liveline`, `price-ticker`, `top-movers`, `coin-movers`, `market-hours`, `bitcoin-dominance`, `market-cap-treemap`
  - `crypto` — Crypto & On-chain: `tvl-treemap`, `dex-volume-treemap`, `dex-volume-chart`, `protocol-tvl-treemap`, `protocol-tvl-chart`, `protocol-fees-treemap`
  - `bitcoin` — Bitcoin Network: `btc-fees`, `btc-mempool`, `btc-blocks`, `btc-hashrate`, `btc-difficulty`, `mining-pools`, `lightning-stats`
  - `derivatives` — Derivatives & Options: `funding-rate-chart`, `funding-heatmap`, `open-interest`, `options-put-call`, `options-iv`, `options-oi-strike`
  - `macro` — Macro & Rates: `rates-board`, `yield-curve`, `inflation-pulse`, `labor-market`, `national-debt`, `treasury-auctions`, `financial-stress`
  - `equities` — Equities & Filings: `fundamentals`, `filings-feed`, `short-volume`
  - `sentiment` — Sentiment & News: `fear-greed` (zTerminal striped mood bar), `news-feed`
  - `portfolio` — Portfolio: `portfolio-value`, `portfolio-allocation` (holdings donut), `portfolio-holdings`
  - `journal` — Decision Journal: `journal-open`, `journal-log`, `journal-results`, `journal-score`
  - `tools` — Tools & Utility: `clock`, `countdown`, `stopwatch`, `pomodoro`, `session-progress`, `day-meter`, `holiday-calendar`, `market-hours`, `calculator`, `risk-reward`, `returns-projector`, `breakeven`, `checklist`, `rules-card`, `link-grid`, `daily-analysis`
  - `layout` — Layout & Media: `heading`, `note`, `image`, `video`, `quote`, `divider`, `marquee`, `drawdy`
  - `games` — Games: `snake`, `flappy-bird`, `dino-game`, `dice`, `breathing`
  The package also re-exports `AssetLogo`/`assetLogoUrl`/`tickerOf` (keyless CDN logos + HIP-3 prefix stripping) for hosts.
- `packages/cli` — `zframes init | catalogue | lint | serve | snapshot` (tsup-bundled bin that ALSO ships a **prebuilt runtime bundle** — a `vite build` of `apps/runtime` vendored by `scripts/build-runtime.mjs` on `pnpm build:cli`, gitignored under `runtime/`). `init` writes a bare, schema-valid `dashboard.json` (the fixed package.json-style envelope — `version` semver string, `title`, `author`, then `grid` geometry, unicorn `background`, accent `theme`, card-surface `appearance`, empty `frames`) so the agent fills only `frames`, never hand-authoring boilerplate (`--title`/`--author`; refuses to clobber unless `--force`); `serve` is the runtime (`npx zframes serve dashboard.json` → node:http server on 127.0.0.1:37263 serving the bundle + the user's file + a writeback PUT); `lint` is the generating agent's feedback loop; `snapshot` feeds /zframes-brief. **The CLI is the only published artifact** (the `@zframes/*` packages stay `private`, inlined via tsup `noExternal`).
- `skills/zframes/SKILL.md` — the agent skill: resolve CLI (`npx zframes`) → `init` a bare `dashboard.json` → read catalogue → interview → fill `frames` → lint → `zframes serve` (npx-only; no scaffold).
- `apps/runtime` — the Vite app that IS the runtime. `App.tsx` **fetches** its spec at runtime from `/__zframes/dashboard.json` (served by the dev plugin under `pnpm dev`, by `zframes serve` in prod) and renders it through `DashboardEditor` (not the bare `DashboardRenderer`), so it's end-user-editable; `onSave` PUTs the edited spec to `/__zframes/dashboard` and reloads. Tailwind v4 via `@tailwindcss/vite` (`@source` scans workspace packages). `pnpm build:cli` `vite build`s this app into the CLI's `runtime/`. Renders `spec.background` via `src/background.tsx` (Unicorn Studio scene through `unicornstudio-react`, lazy-loaded, behind a contrast scrim). Renders a pinned live ticker tape via `src/ticker-tape.tsx` (`TickerTape`) — **host chrome, not a frame**: always-on, fixed to the viewport bottom, outside the editable grid, never declared in the spec. It runs inside `<FramesProvider>` (shares the app's providers, no extra WS), surfaces stocks first (`getDayStats(["xyz:*"])`) then the crypto universe (`getDayStats()`), each with a logo + live mid + 24h change, in a CSS marquee.
- `patches/liveline@0.0.7.patch` — vendored from zhive (DPR fix + label precision); applied via pnpm `patchedDependencies`

## Conventions

- pnpm only. Packages ship TypeScript source (`main: src/index.ts`); the runtime's Vite consumes them directly (`optimizeDeps.exclude`).
- Every frame schema field needs `.describe()` — schemas are read by generating agents via `catalogueForAI`.
- `dashboard.json` is the AI-generated artifact. Invalid frame configs render as per-frame error cards (the agent's feedback loop), never crash the dashboard.
- Frame chrome (cards, titles, hover) lives in the renderer's injected `.zf-*` stylesheet, themeable via `--zf-*` CSS vars — frames style only their interior.
- Dashboard background: the **spec declares** it (`background: {type: "gradient"|"unicorn"|"none", projectId?, opacity?}`), the **host renders** it (runtime's `background.tsx`) — same split as providers. Scene `opacity` is kept moderate (~0.15) so it reads as a living backdrop in the gutters, not a distraction; cards are opaque so content always wins. `unicornstudio-react` lives only in the runtime, never in core. **Footgun:** a `unicorn` background loads the SDK from a jsDelivr CDN and the scene from a hosted Unicorn Studio project (`K42KSY4FXeXhjVOj9RgT` is the zframes default) — keyless, but an external runtime dependency (like the Google Fonts import); it falls back to the body gradient if the SDK fails. The skill emits `unicorn` by default.
- Original assets only. Deliberate exceptions (all Zentry's own IP; public release pends owner sign-off): `packages/charts` (port of zTerminal) and `packages/frames/src/use-countdown.ts` (port of zhive's `useCountdown` — the optimized global-tick/viewport-gated readout that drives the `clock` frame).
- Base charts stay generic — no business logic, no data fetching; frames own data, charts own rendering.
- **Provider caching is centralised** — every provider method that hits a free, rate-limited public API wraps its fetch in the shared `@zframes/core/cache` (`TtlCache`), never a hand-rolled `memo`/`Map`/`localStorage` cache. One `TtlCache` instance per logical source; a `key` distinguishes argument variants (series id, CIK, page size), a single-value source uses a constant key (`"latest"`). It gives freshness (TTL), in-flight dedup, stale-on-error, and opt-in `persist` (localStorage; browser-only, JSON-safe values only — not `Map`s) in one place. **Pick the TTL just under the hook's poll interval** so background polls still refresh while reloads/extra frames reuse the cache (e.g. CoinGecko 12 m under a 15 m poll; Treasury 3 h under a 6 h poll). Capability hooks in `packages/core/src/hooks.tsx` own the poll cadence (`usePolled`'s `refreshMs`, jittered); the provider cache owns reuse — keep the two distinct.
- **Editable dashboard** (`@zframes/core/editor` → `DashboardEditor`, GridStack, mirrors Zentry's "customise mode"): the agent generates `dashboard.json` AND a human edits the *same* file — drag/resize/add/delete + a per-frame config rail, then Save. Edits round-trip the human-readable `dashboard.json` (the host's `onSave`), never a localStorage blob — the spec read/write contract lives in `@zframes/core/serve` (`handleSpecRead`/`handleSpecWrite`) and is used by BOTH `@zframes/core/vite`'s `dashboardWriteback()` (dev) and the CLI's `serve` (prod), so the round-trip is identical in both. GridStack owns each item's DOM, so frames render into per-item React roots wrapped in `FramesProvider` (shared provider instances — no duplicate WS). New-frame default config = `buildDefaultConfig(def)` (in `editor.tsx`): all-optional frames resolve from `schema.safeParse({})`, and required-field frames are seeded with schema-valid placeholders (symbol fields → a stocks-first default, required strings → readable placeholders, `.positive()` numbers → 1, min-length arrays → distinct items) so a freshly-added frame renders immediately instead of as an error card. **Footgun:** `@zframes/core/vite` is loaded by Vite's Node config-loader, so it imports the shared module by package subpath (`@zframes/core/serve`), NOT a relative `./serve` — a relative extensionless import fails under Node there.

## Scope

- Keyless by default — the published CLI and all 14 market-data providers are keyless (free public APIs, no key required). An opt-in keyed/account tier exists (`provider-binance` for a connected Binance account, `provider-wallet` for a public on-chain address) but is separate from the default keyless set and not wired into the published CLI.
- Stocks-first — equity perps via Hyperliquid HIP-3 builder dexes (`dex` param, e.g. `xyz:TSLA`), with crypto alongside.

Architecture decisions, the distribution model, and roadmap live in `docs/decisions/` (kept locally, not in the public repo).
